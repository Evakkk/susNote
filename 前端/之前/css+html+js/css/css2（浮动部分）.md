# 前端 css（浮动部分）
###### 传统网页布局的三种方式
```
	普通流（标准流）：标签按照规定好默认方式排列

	浮动

	定位
```	

实际开发中，一个页面基本包含了三种布局方式

* `为什么需要浮动？`
   1. 让多个块级盒子(div水平排列成一行)如何实现？
```
	标准流布局 可以将div转成行内块元素、但是并列会有缝隙距离难以控制。

	产生缝隙的原因：行内块元素之间的换行会产生缝隙

	<div></div><div></div><div></div>	(无缝隙)

	<div></div>
	<div></div>
	<div></div>		（这种写法有缝隙）
```
   2. 实现两个盒子的左右对齐（一左一右）如何实现？
```
	标准流布局 可以实现、但很复杂
```

很多布局效果，标准流很难完成甚至没有办法完成，此时可以利用浮动完成布局。因为浮动可以改变元素标签的默认排序方式。（浮动最典型应用：可以让多个块级元素一行内排列显示）

|
|

* 网页布局第一准则：多个块级元素纵向排列找`标准流`，多个块级元素横向排列找`浮动`
* 网页布局第二准则：先设置盒子大小，再设置盒子位置

* 浮动特性：
```
1.浮动元素会脱离标准流（脱标）
2.浮动的元素会一行内显示并且元素顶部对齐
3.浮动的元素会具有行内块元素的特性

最重要的特性
	1.图片里标准流的控制（浮）移动到指定位置（动）
	2.浮动的盒子！不再保留原先位置！（后面标准流位置会顶上去代替，前面的没有影响）
```

* 浮动元素经常和标准流父级搭配使用
	先用标准流的父元素排列上下位置，之后内部子元素采取浮动排列左右位置

* 父级很多情况下不方便给高度，但是子盒子浮动不占有位置，因此不给高度，父元素高度则会成为0，父元素高度为0会影响下面标准流盒子（对后面的元素排版产生影响）------因此需要清除浮动

* 清除浮动
```
		清除浮动本质：清除浮动元素造成的影响

		如果父盒子本身有高度则不需要清除浮动
		
		清除浮动之后，父级就会根据浮动子盒子自动检测高度。 父级有了高度就不会影响下面的标准流了


选择器{
	clear:属性值;
}

属性值有：left、right、both（一般用both）

清除浮动的策略是：闭合浮动（只让浮动元素在父盒子内部影响，不影响父盒子外面的其他盒子）
```

* 清除浮动的方法
```
1.额外标签法（隔墙法）
		在浮动元素末尾添加一个空的标签(该标签必须为块级元素)
		<div style="clear:both"></div>

2.父级添加overflow属性（一般写hidden）
		值为：hidden、auto、scroll

3.父亲:after伪元素（额外标签法的升级版）给父元素添加（常用）
	.clearfix:after{
		content: "";
		display: block;
		height: 0;
		clear: both;
		visibility: hidden;
	}
	.clearfix {
		*zoom: 1;
	}

	给父元素添加 class="clearfix"	

4.父亲双伪元素清除浮动 也是给父元素添加（常用？）
	.clearfix:before,.clearfix:after {
		content: "";
		display: table;
	}
	.clearfix:after{
		clear: both;
	}
	.clearfix {
		*zoom: 1;
	}

	给父元素添加 class="clearfix"
```

* 实际开发中，我们不会直接用链接a 而是用li 包含链接（li+a）的做法

* 浮动的盒子不会有外边距合并的问题
